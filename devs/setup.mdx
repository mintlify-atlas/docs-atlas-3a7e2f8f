## Prerequisites

Before you start, make sure you have the following installed:

- **Android Studio** (latest stable release) — used for editing, building, and debugging.
- **JDK 17** — CloudStream itself is built with OpenJDK 17 (see `jitpack.yml`). Android Studio bundles a JDK, but verify the version in **File → Project Structure → SDK Location**.
- **Git** — for cloning templates and managing your extension's source.
- **Kotlin knowledge** — extensions are written entirely in Kotlin. Familiarity with coroutines (`suspend` functions) is essential since all network calls are asynchronous.

## Project setup

<Steps>
  <Step title="Create or clone a project">
    The fastest way to get started is to clone the community-maintained extension template repository. It includes a pre-configured Gradle setup, a working plugin scaffold, and a build script that produces `.cs3` files.

    Alternatively, create a new **Android** project in Android Studio (not a Library project — you need the APK output that gets renamed to `.cs3`).

    <Tip>
      Search GitHub for CloudStream extension templates to find community-maintained starters. They save considerable time compared to configuring Gradle from scratch.
    </Tip>
  </Step>

  <Step title="Add JitPack to your dependency resolution">
    CloudStream is published to JitPack, so you need to add it to your repository list. In a modern Gradle project using `settings.gradle.kts`:

    ```kotlin settings.gradle.kts
    dependencyResolutionManagement {
        repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
        repositories {
            maven { url = uri("https://jitpack.io") }
            google()
            mavenCentral()
        }
    }
    ```

    <Note>
      JitPack must be listed before `google()` and `mavenCentral()` to ensure it is consulted when resolving the CloudStream artifact.
    </Note>
  </Step>

  <Step title="Add the CloudStream dependency">
    In your module-level `build.gradle.kts`, add the CloudStream library as a `compileOnly` dependency:

    ```kotlin build.gradle.kts (app/module)
    dependencies {
        val cloudstreamVersion = "pre-release"
        compileOnly("com.github.recloudstream:cloudstream:$cloudstreamVersion")
    }
    ```

    **Why `compileOnly` and not `implementation`?**

    The CloudStream app already includes the library at runtime. If your `.cs3` file also bundled it, you would ship a large duplicate that could conflict with the version the app expects. Using `compileOnly` makes the library available at compile time for type-checking and IDE support, but excludes it from the final output.
  </Step>

  <Step title="Configure the plugin manifest">
    Each `.cs3` file contains a small manifest that tells CloudStream which class to instantiate. This is typically generated by a Gradle task or plugin included in extension templates. The manifest corresponds to the `BasePlugin.Manifest` data class:

    ```kotlin BasePlugin.Manifest fields
    class Manifest {
        var name: String? = null              // Display name of the plugin
        var pluginClassName: String? = null   // Fully-qualified name of your @CloudstreamPlugin class
        var version: Int? = null              // Integer version, increment to trigger auto-updates
        var requiresResources: Boolean = false // Set true if your plugin uses Android resources
    }
    ```

    Extension templates handle writing this manifest into the APK's assets automatically as part of the build. If you are setting up manually, you need a Gradle task that writes a `manifest.json` into `src/main/assets/`.
  </Step>

  <Step title="Annotate your plugin class">
    Your entry point class must be annotated with `@CloudstreamPlugin`. Without it, the loader cannot identify which class to instantiate.

    ```kotlin MyPlugin.kt
    import com.lagradost.cloudstream3.plugins.CloudstreamPlugin
    import com.lagradost.cloudstream3.plugins.Plugin
    import android.content.Context

    @CloudstreamPlugin
    class MyPlugin : Plugin() {
        override fun load(context: Context) {
            registerMainAPI(MyProvider())
        }
    }
    ```

    <Warning>
      Only one class per `.cs3` file should carry the `@CloudstreamPlugin` annotation. Having multiple annotated classes will cause the loader to behave unpredictably.
    </Warning>
  </Step>

  <Step title="Build the extension">
    Run the release build task from the terminal:

    ```bash
    ./gradlew assembleRelease
    ```

    The build output is an APK in `app/build/outputs/apk/release/`. Extension templates typically include a Gradle task that copies and renames this file to `<PluginName>.cs3` in a convenient output directory.

    To install and test during development, you can sideload the `.cs3` file directly onto your device using the CloudStream app's local extension installer in **Settings → Extensions**.

    <Tip>
      Use the CloudStream app's built-in provider testing tool under **Settings → Extensions → [your extension] → Test** to run your provider's methods interactively without writing a test harness.
    </Tip>
  </Step>
</Steps>

## Gradle configuration summary

Here is a complete minimal `build.gradle.kts` for an extension module, showing all relevant settings in one place:

```kotlin build.gradle.kts (complete example)
plugins {
    id("com.android.application")
    id("org.jetbrains.kotlin.android")
}

android {
    compileSdk = 34
    defaultConfig {
        minSdk = 21
        targetSdk = 34
    }
    compileOptions {
        sourceCompatibility = JavaVersion.VERSION_17
        targetCompatibility = JavaVersion.VERSION_17
    }
    kotlinOptions {
        jvmTarget = "17"
    }
}

dependencies {
    val cloudstreamVersion = "pre-release"
    compileOnly("com.github.recloudstream:cloudstream:$cloudstreamVersion")
}
```

```kotlin settings.gradle.kts (complete example)
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        maven { url = uri("https://jitpack.io") }
        google()
        mavenCentral()
    }
}

rootProject.name = "MyExtension"
include(":app")
```
